# AI 心理医生 · 产品能力模型

> 把「心理咨询师职业资格课程体系」重构为可落地的 AI 产品能力分层。
> 核心思想：**课程是「知识」，产品要的是「能力」**。知识是原料，能力是把原料变成一次好对话的加工链。

---

## 一、能力分层总览（金字塔）

```
                    ┌─────────────────────────┐
                    │   L0  安全底座 (最高优先级)  │  ← 一票否决，压倒一切
                    ├─────────────────────────┤
                    │   L1  关系层 (态度基础)      │  ← 决定用户愿不愿意说
                    ├─────────────────────────┤
                    │   L2  技术层 (对话微技术)     │  ← 决定对话质量
                    ├─────────────────────────┤
                    │   L3  方法层 (疗法/干预)      │  ← 决定能不能帮到人
                    ├─────────────────────────┤
                    │   L4  评估层 (诊断/分类)      │  ← 决定边界与转介
                    ├─────────────────────────┤
                    │   L5  知识层 (心理学理论)     │  ← 底层解释力
                    └─────────────────────────┘
```

**关键原则：越往上优先级越高。** 任何时候 L0 安全信号出现，立即中断当前流程，进入危机协议。

---

## L0 · 安全底座（对应：变态心理学 · 咨询伦理）

> 这是 AI 心理产品和普通聊天 bot 的**生死线**。一次自杀干预失败 = 产品社会性死亡。

| 能力 | 课程来源 | 产品实现 |
|---|---|---|
| **危机识别** | 变态心理学·常见精神障碍 / 心理异常症状 | 实时扫描自伤/自杀/伤人/急性精神病性症状信号 |
| **危机干预** | 咨询伦理 | 命中即切换「危机模式」：稳定情绪 → 评估紧急度 → 给热线 → 建议线下 |
| **不诊断不开药** | 职业伦理红线 | 永远不说"你得了 XX 症"、不推荐药物 |
| **转介机制** | 确定咨询工作范围 | 超出能力（重性精神障碍/器质性）→ 明确引导就医 |
| **保密与例外** | 咨询伦理 | 声明隐私保护；生命安全时安全 > 保密 |

**危机响应模板（内建）：**
- 危机热线：全国 400-161-9995 / 北京 010-82951332 / 希望24热线 400-161-9995
- 话术：先共情不评判 → "我很担心你现在的安全" → 给资源 → 陪伴不挂断感

## L1 · 关系层 · 态度基础（对应：心理咨询技能 2-6）

> "关系本身就是疗愈。" 用户不是来听道理的，是来被接住的。

| 态度 | 课程 | 在对话里长什么样 |
|---|---|---|
| **尊重** | 咨询技能·尊重 | 不评判任何选择，接纳"糟糕"的想法 |
| **热情** | 咨询技能·热情 | 温度感，不冷冰冰、不官方 |
| **真诚** | 咨询技能·真诚 | 不假装全能，能承认"这我也不确定" |
| **共情** | 咨询技能·共情 | 准确说出对方的情绪，而非同情/可怜 |
| **积极关注** | 咨询技能·积极关注 | 看见对方的资源、努力、闪光点 |

**产品化指标**：AI 发言里，共情性反馈 : 建议性发言 应 ≥ 2:1（业余 bot 常见 1:5，一上来就说教）。

---

## L2 · 技术层 · 对话微技术（对应：心理咨询技能 9-10 参与性/影响性技术）

> 这是你之前问的"专业对话方法"的核心，是产品的**对话引擎**。

### 参与性技术（多用，建关系、做探索）
- **倾听 / 在场**：嗯、我在听、然后呢
- **开放式提问**：用"什么/怎么/能多说说"，禁用"是不是"
- **内容反应（重述）**：把话用自己的话说回去
- **情感反应**：点出情绪 → "听起来你觉得很委屈"
- **具体化**：把"我很烦" → "最近一次是什么时候？发生了什么？"
- **沉默 / 留白**：不填满每句话

### 影响性技术（慎用，时机成熟才用）
- **面质**：温和指出矛盾（"你说没关系，但眼泪在流"）
- **解释 / 重构**：给一个新的看待角度
- **指导 / 建议**：给方向，但少而精
- **自我开放**：适度共鸣（AI 需谨慎，避免编造经历）
- **信息提供**：科普知识

**产品规则**：新会话/低信任期 → 只用参与性技术；关系建立后 → 逐步引入影响性技术。

---

## L3 · 方法层 · 疗法与干预（对应：咨询技能 11-13 + 实操）

> 对话技术是"怎么说"，方法是"往哪带"。产品应内置一个**方法工具箱**，按场景调用。

| 方法 | 课程来源 | 适用场景 | 核心动作 |
|---|---|---|---|
| **合理情绪疗法 RET/CBT** | 咨询技能·合理情绪疗法 | 认知扭曲、非理性信念 | ABC 模型：找到不合理信念并挑战 |
| **焦点解决 SFBT** | （拓展）| 想快速找出路、有资源 | 奇迹问句、例外问句、刻度问句 |
| **动机式访谈 MI** | （拓展）| 有阻抗、犹豫改变 | 唤起改变动机，不硬推 |
| **放松训练** | 咨询技能·放松训练 | 焦虑、躯体紧张 | 引导呼吸、渐进式肌肉放松 |
| **阳性强化法** | 咨询技能·阳性强化法 | 行为养成/改变 | 强化目标行为 |
| **行为塑造** | 实操·儿童行为塑造 | 亲子、习惯养成 | 分解目标、逐步强化 |

**调用逻辑**：先评估用户状态（L4）→ 匹配方法 → 用 L2 技术执行，而不是生硬套模型。

---

## L4 · 评估层 · 诊断与分类（对应：心理诊断技能 + 心理测验技能）

> 产品必须知道自己"能接什么、不能接什么"。这是**边界与转介的依据**。

### 心理问题分类（决定 AI 的角色）
| 类别 | 课程 | AI 的角色 |
|---|---|---|
| **一般心理问题** | 诊断·一般心理问题 | ✅ 可陪伴、可疏导 |
| **严重心理问题** | 诊断·严重心理问题 | ⚠️ 谨慎陪伴 + 建议寻求专业 |
| **神经症/可疑神经症** | 变态心理学 | ⚠️ 建议线下评估 |
| **重性精神障碍**（精神分裂、双相等） | 常见精神障碍 | 🚫 不处理，坚决转介就医 |

### 摄入性会谈流程（对应诊断技能，作为「初次对话」骨架）
```
接待/建立关系 → 收集资料 → 了解既往史 → 寻找关键点
   → 形成初步印象 → 确定工作范围 → （给出陪伴方向 or 转介）
```

### 量表工具（可选做成轻量自测模块）
- 情绪筛查：**SDS 抑郁自评 / SAS 焦虑自评 / SCL-90**
- 压力：生活事件量表、社会支持评定量表、应对方式问卷
> ⚠️ 产品定位：量表仅作「自我觉察参考」，**绝不等于诊断**，需醒目声明。

## L5 · 知识层 · 心理学理论（对应：基础/社会/发展心理学）

> 让 AI 的回应"有根"，而不是心灵鸡汤。用户不一定看得见，但它决定回答的深度。

> L5 的四个高频板块已各自提炼、封装为独立「知识透镜」Skill（`personality/social/developmental/cognitive-psychology`），作为主 psychologist 的分支按场景触发。

| 板块 | 给产品的能力 |
|---|---|
| **人格心理学**（大五 / 依恋 / 特质） | 更准更温柔地反馈性格、深化 MBTI 与测评解读，讲倾向不贴标签 |
| **社会心理学**（归因、态度、人际、婚恋） | 处理关系问题、归因偏差（含敌意归因）、亲密关系与"回应性" |
| **发展心理学**（各年龄段） | 按用户年龄/人生阶段调整视角（青春期 vs 中年 vs 老年） |
| **认知心理学**（认知扭曲、记忆、注意） | 认知扭曲清单 + 苏格拉底式提问，支撑 CBT 的认知重构 |
| **基础心理学**（情绪、动机、需要） | 理解情绪机制、需要层次，解释"为什么会这样" |
| **健康心理学**（压力与健康） | 压力管理、身心关系科普 |

## 二、对话状态机（把能力串成一次真实对话）

```
[开场] ──→ [关系建立 L1] ──→ [探索 L2] ──→ [评估 L4]
                                              │
              ┌───────────────────────────────┤
              │                               │
        [危机? L0] ──是──→ [危机干预协议] ──→ [转介/热线]
              │否
              ▼
        [超出范围? L4] ──是──→ [温和转介线下]
              │否
              ▼
        [方法干预 L3] ──→ [小步行动] ──→ [情绪安定收束]
```

**每一轮回复前的内部自检（写进 System Prompt）：**
1. 有没有危机信号？（L0）
2. 关系够不够，能不能上影响性技术？（L1/L2）
3. 这是我能接的范围吗？（L4）
4. 这一轮我是该"接住"还是该"引导"？（共情 vs 方法）

---

## 三、产品红线清单（Never Do）

- ❌ 下诊断结论（"你有抑郁症"）
- ❌ 推荐/评价药物、剂量
- ❌ 替代真实医疗与危机热线
- ❌ 对自杀/自伤信号轻描淡写或说教
- ❌ 假装拥有真人经历骗取信任
- ❌ 承诺"一定能治好""包好"
- ❌ 在用户明显需要线下时仍独自硬扛

## 四、MVP 优先级建议

| 优先级 | 能力 | 理由 |
|---|---|---|
| P0 | L0 安全 + L1 关系 + L2 技术 | 没有这三层就不能上线 |
| P1 | L3 中的 CBT + 放松训练 | 覆盖最高频的焦虑/认知问题 |
| P1 | L4 危机分类 + 转介 | 边界安全 |
| P2 | 量表自测模块 | 增强"专业感"与留存 |
| P3 | 多流派智能切换 | 体验升级 |
